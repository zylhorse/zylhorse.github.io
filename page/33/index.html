概念

设计模式是在面向对象开发设计中反复出现问题的解决方案的总结。

设计原则

  1. 对接口编程而不是对实现编程
  2. 优先使用对象编程而不是继承

设计模式原则

  1. 针对接口编程而不是对实现编程
  2. 优先使用对象组合而不是继承

开闭原则（Open Close Principle）

对扩展开放， 对修改关闭

为使程序的扩展性好，易于维护和升级， 在程序需要进行扩展的时候，不去修改原有代码，使用接口和抽象类。

任何基类可以出现的地方， 子类都可以出现。

编译器为什么要对代码做自动优化

2013-11-07

zylhorse

5分钟

概括

  1. 高级编程语言提供的函数、条件语句、循环这样的抽象编程结构极大的提高了编程效率，然而，这也潜在的使性能显著下降。
  2. 编译器尝试自动优化代码以提高性能，编译器可以转化循环、条件语句和递归函数、消除代码快和利用目标指令集的优势让代码变得高效而简洁。
  3. 因此对于程序员来说写出可读性高的代码比手动优化后变得神秘而难以维护更加可贵。事实上手工优化的代码可能让编译器难以进行额外和更加有效的优化。
  4. 比起手动优化代码，程序员更应该考虑程序设计的各个方面，比如使用更快算法，引入线程级并行机制和框架特性。

内存溢出的场景及解决方案

2013-11-03

zylhorse

5分钟

栈溢出

压栈内存超出系统上限，导致溢出

  1. 函数递归调用层次较深
  2. 局部变量体积过大

解决办法

  1. 将递归使用循环语句实现
  2. 进行尾递归优化： 这样执行的递归函数指挥占用一个栈帧，不会引起栈溢出
  3. 增加系统对进程栈分配大小
  4. 使用堆内存， 使用全局变量

尾递归

  1. 仅在函数的尾位置调用自身
  2. 通过优化，使得计算仅占产量栈空间；
常规递归：
function fact(n) {
    if (n <= 0) {
        return 1;
    } else {
        return n * fact(n - 1);
    }
}
