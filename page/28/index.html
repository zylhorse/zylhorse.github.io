锁

在多线程编程中， 对共享资源竞争的保护机制；

锁缺点

  1. 引起线程阻塞
  2. 申请和释放锁， 增加线程的调度切换， 影响系统性能
  3. 容易造成死锁和活锁等问题
  4. 优先级反转
  5. 不公平
  6. 效率低

公平锁

在锁上等待时间最长的线程优先获取锁的使用权

非公平锁

线程获取锁的使用权是随机的， cpu时间片轮到哪个线程， 哪个线程就获取到锁；

锁类型

  1. 自旋锁（spin lock） ： 自旋锁如果已经被其他线程获取，则调用者就一直循环等待；自旋锁时非阻塞锁， 循环等待时会不断消耗cpu， 不会使线程挂起；
  2. 互斥锁（mutex lock）： 互斥锁被其他线程获取， 则调用者会阻塞线程， 线程被挂起， 不会消耗cpu；…

概述

i/o 多路复用:在单线程中通过记录跟踪多个sock 状态， 来同时管理多路I/O 流。用以提高服务器的吞吐能力

select

  1. select会修改传入的参数数组
  2. 任何sock状态改变，select只返回并不告诉是哪个sock状态改变，需要手动遍历
  3. 最高监听1024个sock连接；
  4. 线程不安全，如果在其他线程关闭sock会导致未知错误

poll

  1. poll 修复很多select 问题， 不在修改参数数组；不在限制sock连接数量；
  2. poll 仍然线程不安全

epoll

  1. epoll 修复大多数select和poll问题， 线程安全， 并通知哪个sock状态改变；
