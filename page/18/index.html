高可用扩容

  1. 垂直扩容： 提升机器内存配置， 不需要应用程序支持；
  2. 水平扩容： 增加节点，需要应用程序支持；

哨兵模式（sentinel）

故障转移后redis客户端将无法感知redis节点的变化；导致无法正常使用；

哨兵模式高可用客户端流程

  1. 客户端获取 sentinel 集合；
  2. 客户端获取master 信息；
  3. 客户端验证获取的master 为真正的master， 防止故障转移期间主节点的变化；
  4. 客户端订阅sentinel节点 相关频道， 时刻获取节点去读写；

bigkey

  1. 单个key 存储value很大
  2. hash set list zset 等存储元素过多

如何发现

  1. redis-cli –bigkeys
  2. rdbtools 分析rdb 文件
  3. redis-rdb-cli 第三方工具

怎么解决

  1. 减少list hash set zset 成员， 对成员过多的数据进行取模分片
