Golang跨平台之交叉编译

2018-01-23

zylhorse

5分钟

概述

交叉编译： 在一个平台生成另一个平台的可执行代码。

golang 交叉编译变量

  1. GOOS:目标系统
  2. GOARCH:目标架构
  3. CGO_ENABLED: 开启cgo

支持的编译平台

条件编译

golang 采用Tag标记和文件后缀方式实现条件编译；

build tag

规则：

  1. build tag必须在文件顶部，可以有多个， 之间是AND关系 ：
// +build linux darwin
// +build arm
  1. 空格位或， 逗号为且， 叹号为非
// +build A,B !C,D
// 翻译： (A&&B) || ((!C) && D)
  1. build tag和package xxx 语句之间需要空行分隔：
// +build linux,arm

package main

文件后缀

规则：

以_$GOOS.go 为后缀的文件只能在指定平台编译，其他平台忽略该文件。

syscall_linux_amd64.go：只在 Linux/amd64 下编译
syscall_windows_386.go：只在 Windows/i386 下编译
syscall_windows.go：    只在 Windows 下编译

cgo 交叉编译

karalabe/xgo

问题：

  1. 不支持 go mod 模式
  2. 不能 (完整支持) 编译本地代码
  3. 不支持最新的 golang 版本
  4. 项目维护热度不足
