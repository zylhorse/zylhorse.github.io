Hugo评论系统-Gitalk

2020-03-07

zylhorse

5分钟

  • 注册GitHub OAuth application

  • 配置config.toml

    [params.gitalk]
  enable = true
  clientId = ""
  clientSecret = ""
  repo = "" # 保存评论的repo
  owner = "" //app 所有者
  admin = "" //app 所有者
  perPage = 20 # 每页评论条数，最大100
  pagerDirection = "last" # 评论排序方式： 最新：last， 最早：first

  • 配置comments.html 在layouts/_default/comments.html中添加一下代码：

    {{- if .Site.Params.valine.gitalk -}}  
  <div id="gitalk-container" class="gcomments"></div>
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/gitalk@1/dist/gitalk.css">
  <script src="https://cdn.jsdelivr.net/npm/gitalk@1/dist/gitalk.min.js"></script>
  <script>
    const gitalk = new Gitalk({
      clientID: {{ .Site.Params.gitalk.clientId }},
      clientSecret: {{ .Site.Params.gitalk.clientSecret }},
      repo: {{ .Site.Params.gitalk.repo }},
      owner: {{ .Site.Params.gitalk.owner }},
      admin: [{{ .Site.Params.gitalk.admin }}],
      id: location.pathname,
      labels: {{ .Params.categories }},
      perPage: {{ .Site.Params.gitalk.perPage }},
      pagerDirection: {{ .Site.Params.gitalk.pagerDirection }},
    });

    gitalk.render('gitalk-container');
  </script>
{{ end }}

  • 配置single.html 在layouts/_default/comments.html中添加一下代码：

    {{ if not ( eq .Params.comments false) }}
       {{ .Render "comments" }}
{{ end }}
  1. Gitalk 会申请授权者所有公共仓库的读写权限
  2. 静态网站会暴露OAuth Application的ID和密钥
  3. Issue labels 会添加location.pathname(本站添加此字段)
