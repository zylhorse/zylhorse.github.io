内存溢出的场景及解决方案

2013-11-03

zylhorse

5分钟

栈溢出

压栈内存超出系统上限，导致溢出

  1. 函数递归调用层次较深
  2. 局部变量体积过大

解决办法

  1. 将递归使用循环语句实现
  2. 进行尾递归优化： 这样执行的递归函数指挥占用一个栈帧，不会引起栈溢出
  3. 增加系统对进程栈分配大小
  4. 使用堆内存， 使用全局变量

尾递归

  1. 仅在函数的尾位置调用自身
  2. 通过优化，使得计算仅占产量栈空间；
常规递归：
function fact(n) {
    if (n <= 0) {
        return 1;
    } else {
        return n * fact(n - 1);
    }
}
展开： 
6 * fact(5)
6 * (5 * fact(4))
6 * (5 * (4 * fact(3))))
// two thousand years later...
6 * (5 * (4 * (3 * (2 * (1 * 1)))))) // <= 最终的展开

尾递归：
function fact(n, r) {
    if (n <= 0) {
        return 1 * r;
    } else {
        return fact(n - 1, r * n);
    }
}
展开：
fact(6, 1) // 1 是 fact(0) 的值，我们需要手动写一下
fact(5, 6)
fact(4, 30)
fact(3, 120)
fact(2, 360)
fact(1, 720)
720 // <= 最终的结果

堆溢出

用户动态分配内存，超出系统内存管理器限制。

解决办法

  1. 限制和固定申请堆内存大小
  2. 禁止申请后不释放
  3. 禁止循环申请内存不中断
