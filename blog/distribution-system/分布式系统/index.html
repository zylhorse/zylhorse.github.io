分布式系统的概念、特性

2019-09-19

zylhorse

5分钟

概念

  1. 建立在网络上的软件系统
  2. 若干独立系统的集合

特性

  1. 分布性： 分布式系统由多个节点组成， 地域上分散；
  2. 自治性： 分布式系统的每个节点都包含自己的处理器和内存， 具备独立处理数据能力；
  3. 并行性： 一个完整的任务可以划分成若干子任务， 运行在分布式系统的多个节点上；
  4. 全局性： 分布式系统中的节点可以互相通信及系统间的相互调用；

CAP

  1. 一致性（consistency）： 分布式系统中的所有数据备份，在同一时刻是否是同样的值（分为弱、强、最终一致性）

  2. 可用性（availability）： 在集群中某一个节点故障后， 集群整体是否还能响应客户端的读写请求（数据高可用性）

  3. 分区容忍性（partition tolerance）： 提供如果在时限内不能达成数据一致性， 必须就当前操作在一致性和可用性之间做出选择；

  4. 任何一个分布式系统都无法同时满足一致性（consistency)、可用性（availability）和分区容忍性（partition tolerance）；

  5. 大多数场景都需要牺牲强一致性来换取系统的高可用性， 往往只需要保证最终一致性；

BASE策略

  1. Basically Available（基本可用）：在分布式系统出现不可预知的故障时， 允许损失部分可用性；
  2. Soft state（软状态）： 允许系统中数据存在中间状态，并认为该中间状态存在不会影响系统的整体可用性；指的是数据不一致；
  3. Eventually consistent（最终一致性）： 系统中所有的数据副本，在一定的时限内，最终能够达到一个一致的状态；
