常见的开源许可协议和注意事项

2014-04-11

zylhorse

5分钟

GPL(GNU General Public License)

GPL 出发点是代码的开源/免费使用和引用/修改/衍生代码的开源/免费使用， 不允许修改/衍生后的代码作为商业软件发布和销售。 代表： linux

BSD

BSD开源协议允许自由的使用/修改源代码， 也可以将修改后的代码作为开源/专有软件发布。但是要满足三个条件：

  1. 源码发布，需要在源码中携带原有代码的BSD协议
  2. 类库/软件发布， 需要在类库/软件的文档和版权声明中包含原有代码的BSD协议
  3. 不可以使用开源代码的作者/机构名字做市场推广

Apache license 2.0

类似BSD协议,Apache license 是对商业应用友好的许可

MIT

类似BSD协议,必须在修改/衍生的代码中保留原许可协议的声明。
